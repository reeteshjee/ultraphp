Introduction to UltraPhP Framework

High Performance

Lightweight architecture with minimal overhead for maximum performance in production environments.

Secure by Design

Built with security best practices to protect applications from common vulnerabilities.

Modular Structure

Flexible component-based design allows for easy customization and extension.

Welcome to the comprehensive documentation for UltraPhP, a lightweight yet powerful MVC framework for modern PHP applications. This guide will help you get started quickly and make the most of the framework's features.

UltraPhP is designed with simplicity and performance in mind, providing developers with the right tools to build robust web applications without unnecessary complexity.

Framework Overview

This documentation covers UltraPhP v1.0, which includes a complete MVC architecture, database abstraction layer, routing system, and helper utilities to accelerate your development process.

Home / Installation

Installation

Getting started with UltraPhP is straightforward. Follow these steps to set up your first project:

System Requirements

  • PHP 7.4 or higher
  • MySQL 5.7+ or MariaDB 10.3+
  • Apache with mod_rewrite enabled or Nginx

Step 1: Download the Framework

Terminal 
git clone https://github.com/username/ultraphp.git my-project
cd my-project

Step 2: Configure Web Server

Point your web server's document root to the public directory of your project. This ensures that only publicly accessible files are exposed to web requests.

Step 3: Install Dependencies (Optional)

Terminal 
composer install

Step 4: Verify Installation

Navigate to your project URL in a web browser. You should see the UltraPhP welcome page if everything is set up correctly.

Quick Tip

For development environments, you can also use PHP's built-in web server:

php -S localhost:8000 -t public
Home / Configuration

Configuration

UltraPhP uses configuration files to manage various aspects of your application. These files are located in the config directory.

Database Configuration

Configure your database connection in the config/database.php file:

config/database.php 
<?php
return [
  'host' => 'localhost',
  'username' => 'dbuser',
  'password' => 'dbpass',
  'database' => 'dbname',
  'driver' => 'mysqli', // or pdo_mysql
  'charset' => 'utf8mb4'
];

The QueryBuilder class will automatically use these settings to establish your database connections.

Security Note

Never commit sensitive database credentials to version control. Consider using environment variables for production environments.

Cache Configuration

Configure caching options in config/cache.php:

config/cache.php 
<?php
return [
  'driver' => 'file', // file, redis, memcached
  'path' => STORAGE_PATH . '/cache',
  'lifetime' => 3600, // seconds
  'prefix' => 'ultra_'
];
Home / Core Concepts

Core Concepts

UltraPhP follows the Model-View-Controller (MVC) architectural pattern to separate concerns and maintain clean, organized code.

Routing

Routing maps URLs to specific controller methods. UltraPhP parses the URL to determine which controller and method to invoke.

URL Structure

Example URL: http://example.com/home/index
This maps to the Home controller and the index method.

Example basic routing file:

routes.php 
// routes.php
$routes = [
  '' => 'Home@index',           // default route
  'home' => 'Home@index',
  'home/landing' => 'Home@landing',
  'about' => 'Home@about',
];

The router parses the URI and calls Home::index(), etc.

Controllers

Controllers handle request logic. They extend the base controller Core\Ultra and define methods as action handlers.

app/Controllers/Home.php 
<?php
namespace App\Controllers;
use Core\Ultra;

class Home extends Ultra {

  public function __construct() {
    parent::__construct();
  }

  public function index($params = []) {
    $data['latest'] = $this->model('NewsModel')->latest();
    view('home', $data);
  }

  public function about() {
    view('about');
  }
}

Models

Models interact with the database, encapsulating data queries. UltraPhP provides a QueryBuilder class for fluent database interactions.

app/Models/NewsModel.php 
<?php
namespace App\Models;
use Core\QueryBuilder;

class NewsModel {
  private $db;
  private $table = 'posts';

  public function __construct() {
    $this->db = new QueryBuilder();
  }

  // SELECT example: Get latest posts
  public function latest($limit = 10) {
    return $this->db
      ->select('id, title, published_on')
      ->from($this->table)
      ->where('published', 1)
      ->orderBy('published_on', 'DESC')
      ->limit($limit)
      ->get();
  }

  // INSERT example: Add a new post
  public function insertPost($data) {
    return $this->db->insert($this->table, $data);
  }

  // UPDATE example: Update a post by ID
  public function updatePost($id, $data) {
    return $this->db
      ->update($this->table, $data)
      ->where('id', $id)
      ->execute();
  }

  // JOIN example: Get posts with category slug
  public function getPostsWithCategory() {
    return $this->db
      ->select('posts.id, posts.title, categories.slug')
      ->from($this->table)
      ->join('categories', 'posts.category = categories.id', 'left')
      ->where('posts.published', 1)
      ->orderBy('posts.published_on', 'DESC')
      ->get();
  }

Views

Views contain the HTML and presentation logic of your application. UltraPhP uses a simple template system with .php view files.

app/Views/home.php 
<!DOCTYPE html>
<html>
<head>
  <title>Welcome to UltraPhP</title>
</head>
<body>
  <h1>Welcome to UltraPhP Framework</h1>
  
  <h2>Latest News</h2>
  <ul>
    <?php foreach($latest as $item): ?>
      <li><?= $item->title ?></li>
    <?php endforeach; ?>
  </ul>
</body>
</html>

To render a view from a controller:

Example View Usage 
// Inside a controller method
public function index() {
  $data = [
    'title' => 'Homepage',
    'items' => $this->model('ItemModel')->getAll()
  ];
  
  view('home', $data); // Loads app/Views/home.php with $data
}

Helpers & Functions

UltraPhP includes several helper functions to make common tasks easier:

Function Description Example
url($path = '') Generate a URL to an application resource url('assets/css/style.css')
view($name, $data = []) Load and render a view file view('user/profile', ['user' => $user])
redirect($url) Redirect to another URL redirect('home')
session($key, $value = null) Get or set session data session('user_id', 123)
e($string) HTML escape a string e($userInput)

Custom Helpers

You can create your own helper functions in app/Helpers/ directory. They will be automatically loaded by the framework.

Using Libraries

Example Library Usage 
// Load and use the Upload library
$upload = $this->library('Upload');

$config = [
  'upload_path' => 'uploads/',
  'allowed_types' => 'jpg|png|gif',
  'max_size' => 2048 // 2MB
];

$upload->initialize($config);

if ($upload->do_upload('file_field')) {
  $data = $upload->get_data();
  echo "File uploaded: " . $data['file_name'];
} else {
  echo $upload->display_errors();
}

Usage Examples

Creating a Simple CRUD Application

Here's a complete example of a simple CRUD application for managing posts:

1. Create the Model

app/Models/PostModel.php 
<?php
namespace App\Models;
use Core\ModelBase;

class PostModel extends ModelBase {
  protected $table = 'posts';
  
  public function getAll() {
    return $this->db->table($this->table)->get();
  }
  
  public function create($data) {
    return $this->db->table($this->table)->insert($data);
  }
  
  public function find($id) {
    return $this->db->table($this->table)->where('id', $id)->first();
  }
  
  public function update($id, $data) {
    return $this->db->table($this->table)->where('id', $id)->update($data);
  }
  
  public function delete($id) {
    return $this->db->table($this->table)->where('id', $id)->delete();
  }
}

2. Create the Controller

app/Controllers/Post.php 
<?php
namespace App\Controllers;
use Core\Ultra;

class Post extends Ultra {
  private $postModel;
  
  public function __construct() {
    parent::__construct();
    $this->postModel = $this->model('PostModel');
  }
  
  public function index() {
    $data['posts'] = $this->postModel->getAll();
    view('posts/index', $data);
  }
  
  public function create() {
    if ($_SERVER['REQUEST_METHOD'] === 'POST') {
      $postData = [
        'title' => $_POST['title'],
        'content' => $_POST['content'],
        'created_at' => date('Y-m-d H:i:s')
      ];
      
      $this->postModel->create($postData);
      redirect('post');
    }
    
    view('posts/create');
  }
  
  public function edit($id) {
    if ($_SERVER['REQUEST_METHOD'] === 'POST') {
      $postData = [
        'title' => $_POST['title'],
        'content' => $_POST['content'],
        'updated_at' => date('Y-m-d H:i:s')
      ];
      
      $this->postModel->update($id, $postData);
      redirect('post');
    }
    
    $data['post'] = $this->postModel->find($id);
    view('posts/edit', $data);
  }
  
  public function delete($id) {
    $this->postModel->delete($id);
    redirect('post');
  }
}
Home / .htaccess Setup

.htaccess Setup

To use UltraPhP's clean URL routing, you'll need an appropriate .htaccess file in your public directory:

public/.htaccess 
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ index.php?url=$1 [QSA,L]

For Nginx, add this to your server block:

nginx.conf 
location / {
  try_files $uri $uri/ /index.php?url=$uri&$args;
}

Important

Make sure your web server is properly configured to handle these rewrite rules. For Apache, make sure mod_rewrite is enabled.
